# Docs Index

This directory is the system of record for the project.

## Sections

- `architecture/`: package boundaries, system decomposition, runtime model
- `contracts/`: formal interfaces, invariants, and data boundaries
- `design-docs/`: engineering principles and agent-facing design guidance
- `exec-plans/`: active and completed execution plans with decision logs
- `product-specs/`: scope, milestones, and user-facing goals
- `references/`: external sources that should be translated into repo-local knowledge
